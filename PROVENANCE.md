# Provenance of every number in `mwm_audit_paper.tex`

Audited 2026-07-23. This file records which artifact each paper number comes
from, and which script regenerates it. `experiments/check_paper_numbers.py`
enforces all of it and exits non-zero on any mismatch (66 checks).

```
python experiments/check_paper_numbers.py
```

## The checkpoints under audit

`experiments/checkpoints_long/{gold,eurusd,usdjpy}/best_model.pt`, trained
2026-07-16/17 on a single Colab GPU, 60 epochs each, one instrument per session
(`experiments/MWM_colab_retrain.ipynb`). Best validation loss at epochs
51 / 48 / 50; `loss_history.json` alongside each checkpoint holds all 60 epochs
of train/val total, pred and SIGReg loss.

Every number in the paper is computed from **these** checkpoints on the range
each encoder was trained on (`o1b_boundary_slide.CACHED_RANGE`: gold
2015-01-01..2026-07-01, EUR/USD and USD/JPY 2020-01-01..2026-07-01), with
date-based splits from `retrain_long.REGIME_SPLITS`.

## Known limitations of the evidence base

Recorded here rather than left implicit.

1. **`experiments/checkpoints/` is superseded.** That directory holds the older
   short-history checkpoints and the live-trading heads. No paper number comes
   from it. The paper's checkpoints are in `checkpoints_long/`.

2. **`surprise_timeseries_long_*.json` and `event_precision_recall_results.json`
   (dated 2026-07-01) are stale.** They were produced from the superseded
   checkpoints and from a longer data range than the encoders were trained on.
   They are kept for history. The current equivalents are
   `surprise_timeseries_matched_*.json` and
   `event_precision_recall_matched.json`. The paper cites only the latter.
   Recall moved from 0.50/0.34/0.40 (stale) to 0.15/0.25/0.26 (current), which
   strengthens the paper's claim that S_t is not a calendar detector.

3. **The SIGReg finite-sample floor (0.0013 at N=400)** is documented in
   `experiments/RESEARCH_STATUS_2026-07-17.md` but has no standalone result
   file. It is a property of the estimator, not of a checkpoint.

4. **The "verified separately" sentence in the Discussion is the weakest-sourced
   claim in the paper.** It cites a frozen-early-encoder slide and a
   retrain-per-boundary slide. `o1b_frozen_slide_eurusd.json` exists for EUR/USD
   only; `o1b_boundary_slide.py` has no saved result file in this repo. Both
   scripts are released and runnable, but that sentence is not reproducible from
   stored artifacts the way every other claim is.

5. **`data/cache/` is included** so the audit reruns offline. The combined
   per-range caches under `data/cache/long/` are the ones the scripts read;
   the loose Yahoo parquets are older pulls kept for the live-trading path.

## Rebuilding the paper

```
python experiments/embedding_spread.py all
python experiments/probe_absolute.py all
python experiments/o3_stratified.py all --dump-arrays
python experiments/regen_surprise_matched.py all
python experiments/o1b_artifact_check.py gold      # and eurusd, usdjpy
python experiments/make_o3_figure.py
python experiments/make_o1b_figure.py
pdflatex mwm_audit_paper.tex && pdflatex mwm_audit_paper.tex
python experiments/check_paper_numbers.py
```

All of it runs on CPU from the released checkpoints and cached data. Only the
checkpoints themselves needed a GPU.
